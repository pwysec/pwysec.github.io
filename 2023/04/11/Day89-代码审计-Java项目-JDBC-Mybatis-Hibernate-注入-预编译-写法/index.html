#Javaweb-数据库操作-模式&写法&预编译等

在Java中执行SQL语句一般有以下几种方式：

-JDBC 注入分析
安全写法（SQLDemo3）： “select * from user where id = ?”;

不安全写法（SQLDemo）： “select * from user where id =”+req.getParameter(“id”);

不安全写法（SQLDemo2）：即使采用了预编译prepareStatement

-Mybatis 注入分析

#号会对语句进行预编译

${ } 只是进行string 替换，动态解析SQL的时候会进行变量替换

安全写法（）： select * from user where name = #{name}

不安全写法（UserDao.xml）：select * from user where name = ${name}

-Hibernate 注入分析
安全写法（）：参数绑定预编译

Query<User>.query=session.createNativeQuery("select * from user where name=:name");

query.setParameter("name",parameter) ;

不安全写法（User.java）：

Query<User>.query=session.createNativeQuery("select * from user where name="+req.getParameter("id"));

总结：

1、预编译使用不当：

前一句预编译了，又来一句拼接语句and…，后面的可以注入

sql=”select * from user where id = ?”;

sql+=”and username =”+req.getParameter(“username”);

2、直接动态拼接：

“select * from user where id =”+req.getParameter(“id”);

3、order by&like&in查询：(由于这三种不能预编译，所以需要过滤器或自定义过滤)

防御：

能预编译的情况都做预编译，一些特殊无法做预编译的，则过滤用户可控的参数。
